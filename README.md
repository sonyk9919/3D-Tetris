# 3D Tetris

## 기술적 구현 포인트

### 🎮 게임 로직

**2D 그리드 기반 충돌 감지**
- 3D 공간이지만 충돌 감지는 X/Y 좌표만 추출해 2D boolean 배열(`mapGrid`)로 관리
- 블록이 프레임마다 0.05씩 낙하하면서 소수점 좌표가 누적되는데, `Math.round()`로 정수화해 렌더링 좌표계와 게임 로직 좌표계를 분리
- 렌더링은 소수점 기반으로 부드럽게, 충돌 감지는 정수 그리드로 정확하게 처리
- 벽 / 바닥 / 쌓인 블록 충돌을 단일 그리드로 통합 처리

**충돌 사전 검증 패턴**
- 이동/회전 적용 전 블록을 클론해 가상으로 이동시킨 뒤 충돌 여부를 먼저 검증
- 유효한 경우에만 실제 블록에 적용해 잘못된 상태로 전이되는 것을 방지

**애니메이션 중 게임 루프 블로킹**
- 라인 클리어 애니메이션 진행 중 `waitClearAnimation` 플래그로 게임 루프를 일시 차단
- 애니메이션 완료 후 실제 메시의 월드 좌표 기반으로 그리드를 전체 재계산해 상태 불일치 방지

### 🎨 렌더링

**고스트 피스 시뮬레이션**
- 별도 물리 계산 없이 현재 블록을 클론한 뒤 충돌이 발생할 때까지 Y축으로 반복 이동시켜 최종 착지 위치 계산
- `traverse()`로 전체 메시에 반투명 머티리얼을 동적 교체
- `depthWrite: false`로 Z-fighting 방지

**낙하 거리 기반 파티클 강도**
- 하드 드롭 시 낙하 거리를 계산해 파티클 이펙트 강도에 반영
- 높은 곳에서 떨어질수록 더 강한 시각적 피드백 제공

### 🏗️ 설계

**템플릿 메서드 패턴 기반 블록 설계**
- `AbstractBlock`에 공통 로직(회전, 색상, 그룹 관리)을 두고 각 테트로미노 구현체는 좌표 배열만 정의
- 새로운 블록 추가 시 좌표 배열만 작성하면 되는 확장 가능한 구조

## 주요 기능

- **3D 렌더링** — Three.js WebGL 렌더러 기반 아이소메트릭 직교 카메라
- **7가지 테트로미노** — I, O, T, S, Z, J, L 블록, 파스텔 8색 팔레트 적용
- **고스트 피스** — 블록이 떨어질 위치를 반투명으로 미리 표시
- **파티클 이펙트** — 블록 착지 시 충돌 파티클, 라인 클리어 시 폭발 파티클
- **카메라 쉐이크** — 블록 착지 시 화면 진동 피드백
- **점수 애니메이션** — GSAP 기반 색상·크기 애니메이션이 적용된 3D 텍스트
- **라인 클리어 애니메이션** — 행 확장 → 파티클 분산 → 용해 → 블록 낙하
- **배경** — 셰이더 기반 야간 하늘, 절차적 별, 잔디 바닥
- **효과음** — 착지 시 충돌음, 라인 클리어 시 점수 획득음

## 프로젝트 구조

```
src/
├── App.tsx
├── render/
│   └── TetrisScene.tsx         # 메인 3D 게임 씬
├── engine/
│   ├── board/board.ts          # 게임 루프 및 핵심 로직
│   ├── block/                  # 테트로미노 추상 클래스 + 7개 구현체
│   ├── map/map-manager.ts      # 2D 점유 그리드 및 충돌 감지
│   ├── viewport/viewport.ts    # Three.js 씬, 카메라, 렌더러
│   ├── ghost/ghost-manager.ts  # 고스트 피스 위치 계산
│   ├── score/score.ts          # 3D 점수 텍스트 및 애니메이션
│   ├── sound/                  # Web Audio API 사운드 시스템
│   └── utils/                  # 블록 팩토리, 파티클, 배경, 수학 유틸
└── utils/
    └── keyboard-util.ts        # 키보드 입력 처리
```
